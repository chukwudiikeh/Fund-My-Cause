# Internationalization (i18n)

This document covers multi-language support for the Fund-My-Cause Next.js frontend.

## Chosen Solution: `next-intl`

[`next-intl`](https://next-intl-docs.vercel.app/) integrates with the Next.js App Router, supports server and client components, and handles pluralization, date/number formatting, and RTL out of the box.

```bash
cd apps/interface
npm install next-intl
```

## Translation File Structure

```
apps/interface/
└── messages/
    ├── en.json       # English (default)
    ├── ar.json       # Arabic (RTL)
    ├── es.json       # Spanish
    ├── fr.json       # French
    └── zh.json       # Chinese (Simplified)
```

Each file mirrors the same key hierarchy:

```json
// messages/en.json
{
  "nav": {
    "home": "Home",
    "campaigns": "Campaigns",
    "create": "Create Campaign"
  },
  "campaign": {
    "goal": "Goal",
    "raised": "Raised",
    "deadline": "Deadline",
    "contribute": "Contribute",
    "progress": "{percent}% funded"
  },
  "errors": {
    "walletNotConnected": "Please connect your wallet first.",
    "transactionFailed": "Transaction failed. Please try again."
  }
}
```

## Next.js App Router Setup

### `next.config.ts`

```ts
import createNextIntlPlugin from "next-intl/plugin";
const withNextIntl = createNextIntlPlugin();
export default withNextIntl({ /* existing config */ });
```

### `i18n/request.ts`

```ts
import { getRequestConfig } from "next-intl/server";

export default getRequestConfig(async ({ locale }) => ({
  messages: (await import(`../messages/${locale}.json`)).default,
}));
```

### Locale routing — `middleware.ts`

```ts
import createMiddleware from "next-intl/middleware";

export default createMiddleware({
  locales: ["en", "ar", "es", "fr", "zh"],
  defaultLocale: "en",
});

export const config = { matcher: ["/((?!api|_next|.*\\..*).*)"] };
```

URLs become `/en/campaigns`, `/ar/campaigns`, etc.

## Using Translations in Components

```tsx
// Server component
import { useTranslations } from "next-intl";

export default function CampaignCard({ percent }: { percent: number }) {
  const t = useTranslations("campaign");
  return <p>{t("progress", { percent })}</p>;
}
```

## RTL Language Support

Arabic, Hebrew, and Persian are right-to-left. Handle this at the `<html>` element:

```tsx
// app/[locale]/layout.tsx
import { notFound } from "next/navigation";

const RTL_LOCALES = ["ar", "he", "fa"];

export default function RootLayout({
  children,
  params: { locale },
}: {
  children: React.ReactNode;
  params: { locale: string };
}) {
  const dir = RTL_LOCALES.includes(locale) ? "rtl" : "ltr";
  return (
    <html lang={locale} dir={dir}>
      <body>{children}</body>
    </html>
  );
}
```

**Tailwind CSS RTL** — use logical properties so layout flips automatically:

```tsx
// ✅ Logical (works for both LTR and RTL)
<div className="ms-4 pe-2">   {/* margin-inline-start, padding-inline-end */}

// ❌ Physical (breaks RTL)
<div className="ml-4 pr-2">
```

For components that need explicit RTL overrides, use the `rtl:` variant:

```tsx
<ChevronIcon className="rotate-0 rtl:rotate-180" />
```

## Currency and Date Formatting

Use `next-intl`'s built-in formatters — they delegate to the browser's `Intl` API and respect locale conventions automatically.

```tsx
import { useFormatter } from "next-intl";

export function CampaignStats({ raised, deadline }: Props) {
  const format = useFormatter();
  return (
    <>
      {/* XLM amounts — no currency symbol, 2 decimal places */}
      <p>{format.number(raised / 1e7, { minimumFractionDigits: 2 })} XLM</p>

      {/* Deadline date in locale-appropriate format */}
      <p>{format.dateTime(new Date(deadline * 1000), { dateStyle: "long" })}</p>
    </>
  );
}
```

| Locale | Amount display | Date display |
|--------|---------------|--------------|
| `en` | `1,234.56 XLM` | `April 30, 2026` |
| `ar` | `١٬٢٣٤٫٥٦ XLM` | `٣٠ أبريل ٢٠٢٦` |
| `de` | `1.234,56 XLM` | `30. April 2026` |

## Contributing Translations

1. Copy `messages/en.json` to `messages/<locale>.json`.
2. Translate every value; keep all keys identical.
3. Do not translate interpolation placeholders like `{percent}` or `{amount}`.
4. Open a PR with the title `i18n: add <Language> translations`.
5. A maintainer will verify completeness and merge.

**Completeness check** — run before opening a PR:

```bash
node scripts/check-i18n.mjs
```

`scripts/check-i18n.mjs` (add to repo):

```js
import en from "../apps/interface/messages/en.json" assert { type: "json" };
import { readdirSync, readFileSync } from "fs";

const files = readdirSync("apps/interface/messages").filter(
  (f) => f !== "en.json"
);
for (const file of files) {
  const locale = JSON.parse(
    readFileSync(`apps/interface/messages/${file}`, "utf8")
  );
  const missing = Object.keys(en).filter((k) => !(k in locale));
  if (missing.length)
    console.error(`${file} missing keys: ${missing.join(", ")}`);
}
```
